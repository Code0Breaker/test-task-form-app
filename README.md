# Loan Application SPA

Тестовое задание: одностраничное приложение из трёх последовательных форм
(личные данные → адрес и место работы → параметры займа) с валидацией,
сохранением состояния между шагами и финальным модальным окном после
отправки на тестовый API.

## Стек

| Слой               | Технология                       | Обоснование                                                                                                                                                            |
| ------------------ | -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Build              | Vite 5                           | Быстрый dev-сервер, минимальная конфигурация.                                                                                                                          |
| Язык               | TypeScript (strict)              | Требование ТЗ; компилятор ловит ошибки до рантайма.                                                                                                                    |
| UI                 | React 18 + Tailwind CSS 4        | Tailwind через `@tailwindcss/vite` — нулевая конфигурация, JIT, итоговый CSS ~4 KB gzip против ~30 KB у Bootstrap. Утилитарные классы держат стилевой код рядом с JSX. |
| Routing            | react-router-dom 6               | Требование ТЗ. Используется как `<Routes>` + общий layout-роут с `<Outlet context>` — это позволяет лифтить состояние шагов без отдельного state-менеджера.            |
| Forms / Validation | react-hook-form + zod            | Минимум перерендеров (uncontrolled), типобезопасные схемы, единый источник правил валидации (см. `*/schema.ts`).                                                       |
| Маска ввода        | react-imask                      | Промышленная реализация маски `0XXX XXX XXX` — точнее и устойчивее самописной.                                                                                         |
| HTTP               | axios + interceptors             | Единый клиент с baseURL, таймаутом и нормализацией ошибок в перехватчике ответа; в dev-режиме перехватчик запроса логирует исходящие вызовы.                           |
| Состояние черновика | React hooks (useState + Outlet) | Без zustand/redux/react-query: данные лежат в `WizardLayout` и прокидываются через `Outlet context` хуком `useWizard()`.                                               |

## Архитектура

```
src/
├── app/
│   ├── WizardLayout.tsx        # источник правды (useState) + <Outlet context>
│   └── router.tsx              # маршруты + StepGuard
├── features/
│   ├── personalInfo/           # форма 1: schema.ts + PersonalInfoForm.tsx
│   ├── addressWork/            # форма 2: + useProductCategories.ts
│   └── loanParams/             # форма 3: + useSubmitApplication.ts
├── shared/
│   ├── api.ts                  # axios instance + интерсепторы
│   ├── constants.ts            # ROUTES, маска, диапазоны слайдеров
│   ├── format.ts               # formatCurrency, humanizeSlug
│   ├── ui.tsx                  # StepShell, Field, SuccessModal, классы кнопок/инпутов
│   └── useWizard.ts            # типизированный useOutletContext()

├── main.tsx
└── vite-env.d.ts
```

### Ключевые архитектурные решения

- **Hook-only state.** Состояние черновика держит `WizardLayout` через
  `useState`. Дочерние шаги (`features/*`) читают и обновляют его хуком
  `useWizard()` — тонкой обёрткой над `useOutletContext`. Никаких
  zustand/redux/jotai: ответственность за состояние локализована в одном
  layout-роуте, тестировать его проще, бандл легче.
- **Step-gating через `<StepGuard>`.** Маршруты `/step-2` и `/step-3` обёрнуты
  в guard, который редиректит назад, если пользователь пытается попасть туда
  вручную, не пройдя предыдущие шаги.
- **Кэш категорий — без библиотеки.** `useProductCategories` использует
  module-level переменные `cache` и `inflight`: первый вызов делает реальный
  запрос, параллельные подписчики дожидаются того же промиса, повторные
  монтирования отдают данные синхронно из кэша. Это полностью покрывает
  требование ТЗ «переиспользования» без введения React Query.
- **axios + interceptors.** Запросный перехватчик логирует исходящие вызовы
  только в dev. Ответный перехватчик нормализует ошибки: вытаскивает
  `response.data.message` или `error.message`, оборачивает в обычный `Error`
  с прикреплённым `status` — фичевые хуки (`useSubmitApplication`,
  `useProductCategories`) ловят однотипные ошибки.
- **Минимальная декомпозиция UI.** Вместо отдельных компонентов
  `TextField/SelectField/...` — один визуальный примитив `<Field>` (label +
  children + error) и пара классовых констант (`inputClass`, `btnPrimary`,
  `btnSecondary`). Сами `<input>`/`<select>`/`<input type="range">` живут
  прямо внутри форм. Это убирает лишний слой абстракции и делает каждую форму
  читаемой сверху вниз.
- **Schema-first валидация.** Каждая форма имеет рядом `schema.ts` (zod) —
  единственный источник правды о правилах валидации и типах значений (`infer`).
  RHF подключает их через `zodResolver`.

## Соответствие ТЗ

- TypeScript, React, React Router — использованы.
- Tailwind для стилизации (по ТЗ Bootstrap «можно использовать», на ваше усмотрение).
- Все поля обязательны; ошибки показываются под полями (zod через RHF).
- Маска телефона `0XXX XXX XXX` (react-imask) + жёсткая регулярка в zod.
- Источник профессий — `GET https://dummyjson.com/products/categories`,
  результат закэширован module-level промисом и переиспользуется при
  повторных монтированиях формы.
- Слайдеры суммы ($200–$1000, шаг 100) и срока (10–30 дней, шаг 1) — нативный
  `<input type="range">` под управлением RHF.
- Отправка `POST https://dummyjson.com/products/add` с
  `{ title: firstName + ' ' + lastName }` (axios).
- После ответа — модалка вида
  «Поздравляем, &lt;Фамилия&gt; &lt;Имя&gt;. Вам одобрена &lt;сумма&gt; на &lt;срок&gt; дней.»
- «Назад» сохраняет введённые данные (state в layout-роуте).
- Данные доступны на любом шаге через `useWizard()`.

## Как запустить

Требования: Node.js 18+ и npm.

```bash
npm install
npm run dev
```

Откройте указанный Vite адрес (по умолчанию <http://localhost:5173>; если
порт занят — Vite сам выберет следующий). Стартовая страница редиректит на
`/step-1`.

### Прочие команды

```bash
npm run typecheck   # tsc --noEmit
npm run build       # production-сборка (tsc -b && vite build)
npm run preview     # отдать собранный dist
```

## Как проверить

1. **Шаг 1.** Нажмите «Далее» на пустой форме — увидите ошибки. Введите
   телефон не по маске — появится сообщение. С валидными данными переход на
   шаг 2.
2. **Шаг 2.** Подождите загрузку списка профессий (запрос идёт на dummyjson).
   Нажмите «Назад» — данные шага 1 на месте. Возвращайтесь на шаг 2 — список
   уже в кэше, во вкладке Network повторного запроса нет.
3. **Шаг 3.** Подвигайте ползунки. «Подать заявку» — в Network увидите
   `POST /products/add`. После ответа появится модалка с подставленными
   данными. После закрытия состояние сбрасывается, пользователь возвращается
   на шаг 1.
4. **Прямая навигация.** `/step-3` без пройденных предыдущих шагов
   редиректит на нужный шаг (`StepGuard`).
